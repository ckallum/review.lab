The two seams that make the parallelism real

Both are hard Wave-1 exit gates, not nice-to-haves. Without them the back half of the plan serializes through one constructor and one undefined data shape.

  • Route-registration seam in createApp. It registers only /health today; six tickets (T1.5, T1.9, T1.10, T2.5, T4.1, T4.3) need data routes. Thread a transaction-owning data layer in and register routes via per-feature src/api/*.ts modules — or every parallel pairing in Waves 3+ collapses.
  • src/api/types.ts — the hydration contract. T1.5 writes the POST, T1.7 hydrates HTML, T1.9 builds the GET JSON — three tickets, one shape that design.md never specs. Pin it in Wave 1 or T1.7's Wave-2 parallelism is false (it renders against a guess and gets reworked).
  • Frontend: web/panels/*.js loader. Eight tickets touch the UI — a thin panel-loader convention lets each add a panel file + one import line instead of all colliding on web/index.html.

Parallelism — and the false-parallelism traps

  • T2.6 pulled forward to Wave 2. Its only hard deps (T1.4, T1.5) are done by end of Wave 1, and it never touches anthropic.ts — running it early also lands the FR-P0.4 generated-file exclusion before T2.1/T2.2 need it.
  • Trap: T1.9a → T1.9b is sequential (write-back depends on read); T3.1 → T3.2 is sequential (postinstall depends on the skill).
  • Trap: the Wave-5 LLM chain must serialize — T1.11/T2.2/T2.5/T2.7 share src/llm/anthropic.ts. Only T2.6 and T2.7's non-hook work escape it.
  • Trap: T3.6 / T4.2 / T4.3 are not one parallel wave — independent backends, but they unblock at very different times.
  • Trap: T1.6's revision-diff fixtures partially serialize behind T1.5 (they need the diff_hash interface) — its 30+ hash-property cases run now; the 10+ diff fixtures wait on the pinned contract.

Speed-of-execution levers (ranked)

  1. Demo file-based first; push the whole Phase-2 LLM column off the first-demo path (~8 tickets). T1.8 gives story-shaped chapters with ANTHROPIC_API_KEY unset — the spec's designed fallback. De-risks the loop before paying any LLM latency. Tradeoff: chapters are structural, not narrative.
  2. Defer the Session Bay cluster (T3.3–T3.7) — ~a full week, all P1, none on the path. Caveat: T3.3 also gates T4.1 ask-the-author.
  3. Cut the LLM chunker T2.3 + its gate T2.4; repoint T2.5 directly at T2.1. Replace the 150k-token path with a loud "diff too large" error + one test.
  4. Merge T3.1+T3.2 and pull to Wave 4 — moves the /publish-review entry point early. Does not move dogfooding (T2.8 is hard-gated on T2.5).
  5. Reduce T2.6 from L to S — generated-file detection + blanket "human" label; drop blame/trailer/majority-wins for a single-author tool.
